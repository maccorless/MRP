## Glossary
- **EoI** — Expression of Interest: the public-facing portal where media orgs apply for press accreditation consideration
- **PBN** — Press by Number: the phase where NOCs formally allocate their IOC-assigned quotas to specific media organizations
- **ENR** — Extended Non-Rights Broadcaster: broadcasters without Olympic media rights who apply for limited access
- **ACR** — Accreditation system (LA28's system, built on ACR system platform)
- **Common Codes** — Shared organization registry within the ACR system; used across all accreditation categories
- **NOC** — National Olympic Committee (206 worldwide); each manages press accreditation for their country
- **IOC** — International Olympic Committee; owns the process and sets quotas
- **LA28 / OCOG** — Organizing Committee for the 2028 Los Angeles Olympic Games; owns the ACR system. The portal BUILD window is 2026; the Games themselves are 2028.

## Vision

### 10x Check
The 10x version of this isn't just a press registration form — it's an **IOC-owned media credentialing platform** that persists across every Olympic and Paralympic Games edition. By 2032 Brisbane, an accredited New York Times photographer from LA28 doesn't re-apply from scratch; their organization profile carries forward, IOC has a global directory of legitimate press organizations with track records, and rogue applications from non-journalists are caught by pattern matching against historical data. The platform becomes the single source of truth for Olympic press identity worldwide.

### Dream State (12-month ideal — August 2026)
A journalist in any of 206 countries opens olympics.com, finds a press accreditation link in their language (EN/FR/ES), submits their organization's details in under 10 minutes. Their NOC gets a clean dashboard of applicants from their territory, can download a report daily, and submits their Press by Number allocation digitally without touching Excel. The IOC sees everything, intervenes in nothing (unless something looks wrong). Zero Excel spreadsheets. Zero email attachments containing PII. Full audit trail for every Games from LA28 forward.

**Note:** "206 countries" refers to submission capability in EN/FR/ES. It does not mean 206 language packs. Explicit scope: EN, FR, ES only in v1.

## System Architecture

### Two-System Boundary (clarified)
The plan describes **two systems with a data handoff**, not a single unified platform:

```
SYSTEM 1: Media Registration Portal (new, on olympics.com)
  - Public EoI form (media orgs apply)
  - NOC dashboards (per-territory view, CSV/Excel export)
  - IOC read-only dashboard (global view, dedup flags)
  - ENR upload + undertaking signature flow
  ↓ DATA HANDOFF (manual or API TBD per ACR readiness)
SYSTEM 2: ACR / ACR system (LA28's existing accreditation platform)
  - PBN quota management (NOCs allocate quotas to orgs)
  - Common Codes (org registry)
  - Press by Name (individual accreditation — Phase 2, 2027)
```

The PBN module could live in System 1 (standalone web app replacing Excel) or System 2 (ACR module). This is the key architecture decision for the April discovery session.

## Deduplication Rules (explicit)

**Match criteria (all three must match for auto-dedup flag):**
- Organization name (normalized: lowercase, stripped of Ltd/Inc/Corp suffixes, accent-normalized)
- Country of registration (ISO 3166-1 alpha-2)
- Primary email domain (e.g., @nytimes.com)

**Partial match (2 of 3):** Flagged as "possible duplicate" — surfaced to IOC dashboard for manual review.

**IOC action flow on duplicate flag:**
1. IOC sees flagged org pair in dashboard
2. IOC can: (a) Merge organizations → one record persists, (b) Mark as distinct → suppresses future duplicate warnings for this pair, (c) Escalate to NOC for clarification
3. NOC is notified by email if escalated; NOC cannot see other NOCs' data

**Edge cases:**
- Same org applying through multiple NOCs (e.g., international wire service with offices in multiple countries): allowed by design; IOC sees the multi-NOC flag
- Freelancers (no org domain): matched on name + country only; single-field match (name alone) auto-flags for IOC manual review — never auto-deduped. No-domain freelancers always routed through IOC review queue.
- Org name change between Games editions: treated as new org; manual linkage via IOC dashboard

## Authentication & Access Control

**Media org / applicant:**
- Public application form — no account required for initial EoI submission
- Optional: create an account to track application status and receive NOC communications
- Account tied to primary contact email; org can nominate multiple contacts post-approval

**NOC dashboard:**
- NOC administrators provisioned by IOC (not self-registration)
- IOC sends credentials to NOC press attaché (per existing IOC contact database)
- Per-NOC access: can only see applications from their own country
- Multiple NOC administrators per country supported (e.g., head + deputy)

**IOC dashboard (read for application data, write for admin actions):**
- IOC Media Operations team (Lucia, Emma, Randi + additional staff)
- Full global view, all NOCs, all applications
- Write actions: dedup resolution, controversy escalation, report generation, NOC credential management

**ENR-specific:**
- ENR organizations apply via same portal with a separate ENR track
- ENR undertaking: authenticated via account (typed full name + date + checkbox = legal acknowledgment; not cryptographic signature unless legal review determines otherwise)
- **DEPENDENCY:** Legal/compliance review of the undertaking mechanism must begin immediately. DocuSign-grade vs. typed-name has different legal weight. Target: legal sign-off by end of April 2026.

## PII & Data Policy

**Data collected at EoI stage (org-level only — not individual PII):**
- Organization name, type, country, primary contact name + email
- Publication/broadcast details
- No individual journalist data at this stage (per Martyn's guidance — fields not finalized 2 years out)

**PII considerations:**
- Primary contact name + email is personal data → GDPR-applicable for EU-based orgs
- Data stored in IOC-controlled infrastructure (not LA28/OCOG)
- Retention: org profiles retained Games-to-Games (see persistence architecture); individual contact data subject to IOC retention policy (to be confirmed)
- Encryption at rest: required; at transit: TLS 1.3 minimum
- Data residency: IOC must confirm EU/Swiss data residency requirement for olympics.com infrastructure

## Games-to-Games Org Persistence — Schema Notes

The data model must treat **organizations as first-class entities** that outlive a single Games edition:

```
Organization
  id: uuid (stable across Games)
  name: string (multilingual, see Language fields)
  country_code: ISO 3166-1 alpha-2
  primary_domain: string
  org_type: enum (media_print, media_broadcast, enr, agency, freelance)
  games_editions: [edition_id] (list of Games this org has participated in)
  common_code: string (nullable, populated when ACR integration active)
  status: enum (active, inactive, banned, pending_review)

ApplicationRecord
  id: uuid
  org_id: → Organization.id
  games_edition_id: → GamesEdition.id
  noc_country: ISO 3166-1 alpha-2
  submitted_at: timestamp
  status: enum (submitted, under_review, approved, rejected)
  ...
```

## Security (public portal threat model)

**Fake/fraudulent organization applications:**
- Rate limiting on submission form: max 3 submissions per IP per hour
- Email domain blocklist: known disposable email provider domains blocked (open-source blocklist, e.g., disposable-email-domains); MX record check is NOT reliable alone (Mailinator has valid MX)
- CAPTCHA on public form
- Manual IOC flag: any org can be manually flagged as "under review" — flags appear in NOC dashboard before PBN
- Long-term: pattern matching against historical data for known bad actors

**PII protection:**
- No individual journalist PII collected at EoI stage (org-level only)
- Contact email treated as personal data; not shared between NOCs
- NOCs see only their own applicants; zero cross-NOC data leakage possible at data layer (row-level filtering)
- Audit log for all admin actions (IOC dedup resolutions, credential grants, escalations)

**Authentication:**
- Public form: no auth required; optional account creation post-submission
- NOC dashboard: IOC-provisioned credentials; MFA required
- IOC dashboard: IOC SSO (existing IOC identity provider); MFA required
- Session timeout: 30 minutes idle

## ENR Undertaking — Design Fork

Two parallel paths until legal review completes (target: end of April 2026):

**Path A (typed name — simpler):**
- Checkbox: "I agree to the terms of this undertaking"
- Typed full legal name + date
- Timestamp + IP address logged
- PDF receipt generated and emailed to signatory
- Sufficient for most jurisdictions for a contractual undertaking

**Path B (DocuSign-grade — stronger):**
- Integrate DocuSign or equivalent e-signature provider
- Legally equivalent to wet signature in more jurisdictions
- ~3-4 weeks additional build time
- Required if IOC legal determines the undertaking has specific enforceability requirements

Engineers should begin building Path A. Path B adds a module on top without discarding Path A work.
