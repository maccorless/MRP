## Affected Pages/Routes
- `/apply` — EoI form. Test email verification flow, form validation, submission, dedup check
- `/directory` — Public org directory. Test opt-in filtering, no PII exposed
- `/noc` — NOC dashboard. Test territory filtering, quota display, CSV/Excel export, application list
- `/noc/applications/[id]` — Application detail. Test status display, returned-for-edit flow
- `/ioc` — IOC dashboard. Test global view, anomaly cards, NOC summary table
- `/ioc/applications` — All applications. Test dedup queue, resolution flow (distinct/merge)
- `/ioc/export` — ACR export. Test JSON generation, preview screen display
- `/api/applications` — Submission endpoint. Test validation, dedup, idempotency, rate limiting
- `/api/verify-email` — Email verification. Test token generation, expiry, replay rejection

## Key Interactions to Verify
- Email verification → form unlock on `/apply`
- Form submission with dedup flag appearing on IOC dashboard
- NOC can only see their own territory's applications (auth boundary)
- IOC can resolve dedup flag as distinct or merged
- Application returned to applicant → applicant can edit and resubmit
- Export to ACR generates valid JSON → preview screen shows formatted output
- Demo gate blocks all access without shared password

## Edge Cases
- Domain extraction from malformed URLs (no protocol, trailing slashes, subdomains)
- Freelancer with no org domain (dedup should use name + country only)
- NOC with zero applications (empty state dashboard)
- Quota at exact capacity (boundary: 50/50 used)
- Quota exceeded (rejection with clear error)
- Double-submit with same idempotency key (second should be no-op)
- Expired verification token (reject, show re-send option)
- Application in returned state for 30+ days (expiration)
- Export of 10,000+ applications (async job trigger)
- Concurrent org creation race condition (unique constraint)

## Critical Paths
- Full EoI flow: email verify → form → submit → dedup check → NOC sees it → IOC sees it
- Auth boundary: NOC user CANNOT access IOC routes or other NOC data
- Dedup: same domain submitted twice → flagged on second submission → IOC resolves
- Export: IOC exports orgs → JSON file written → preview shows correct data
- Audit: every write action logged → audit_events table has correct entries
