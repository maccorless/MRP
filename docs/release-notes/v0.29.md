# v0.29 Release Notes — LA 2028 Press Registration Portal

**Release window:** 2–4 May 2026
**Audience:** IOC, LA28 ACR, Deloitte delivery team

---

## Shipped

### EoI Application Form (Applicant-facing)

- **ENR programming type message now conditional.** The advisory about ENR programming type on the Publication tab only appears for Non-Media Rights-Holder organisations. Previously it showed for all applicants regardless of org type.
- **Phone number auto-formatting.** Org phone and contact cell phone fields now format automatically as the user types, using international standards (libphonenumber). Works for all country formats; no manual formatting required.
- **Spanish (ES) language option confirmed live.** The ES toggle on the public EoI form was present in earlier builds but not deploying correctly. Now confirmed working alongside EN and FR.

### Admin Header

- **Role badge removed.** The static role label that appeared in the admin header bar is gone. Cleaner header; role is visible via the user menu instead.
- **Username is now a dropdown.** Clicking the username in the admin header opens a menu showing the user's assigned roles (primary role plus any additional roles, e.g. "PRP Admin") and the Sign Out action.

### Admin Language Preference (NOC, IOC, OCOG)

- **Language toggle in the admin header.** NOC, IOC, and OCOG users can switch between EN, FR, and ES from the user menu dropdown.
- **Preference persists across sessions.** Language choice is saved to the user's account in the database and restored automatically on next login. No need to re-select after signing out.
- **Full translation dictionaries created.** Complete EN, FR, and ES string sets (~130 strings each) covering navigation, queue columns, status labels, action buttons, EoI detail, PbN, ENR, Direct Entry, Audit Trail, and error messages.

## In Progress / Coming This Weekend

### Admin Language — NOC page wiring

- Wire the language translations into NOC pages so copy actually switches when the toggle is changed. This covers: nav tabs, EoI queue, queue detail, PbN page, ENR page, Direct Entry page, Audit Trail, and dashboard/home.
- IOC and OCOG portals have the toggle live but page copy stays English until a follow-on release (see Notes below).

### ENR Terminology Rename (stakeholder item)

- Replace "Extended Non-Rights Broadcasters" with "Non-Media Rights-Holders" throughout the admin UI and help pages. The ENR accreditation code itself is unchanged.

### Multi-Sport Tooltip (stakeholder item)

- Add a tooltip to the multi-sport free-text field prompting applicants to break down their sport coverage (e.g., "2 EPs Athletics + 2 EPs Swimming").

### EoI Form Copy + Remove Reserved-Org Block (stakeholder item)

- Update applicant-facing help text to "Apply to the NOC of the country where your organisation is based."
- Remove the reserved-organisation submission block. Currently, certain email domains trigger a hard redirect before the application is submitted. New behaviour: accept the EoI and let the NOC filter it. The reserved-org list and IOC-Direct list remain for admin use; they stop being an applicant gate.

### Dedup Improvements (stakeholder item)

- Skip the email-domain deduplication signal for personal email addresses (Gmail, Outlook, Yahoo, etc.). Personal domains are meaningless as a dedup key.
- Skip email-domain dedup for freelancer org types entirely. Name + country stays the dedup key for freelancers.

### ENR Soft Limit (stakeholder item)

- The 3-organisation ENR submission cap per NOC is currently a hard block. Changing to a soft limit: NOC admins will see a warning when they exceed 3, but submission will not be blocked.

## Notes / Known Gaps

- **IOC and OCOG page translations deferred.** The language toggle is live for IOC and OCOG users, but page copy on those portals will stay in English until the follow-on release. NOC pages are the priority for v0.29.
- **PRP Admin is English-only.** No language toggle for PRP Admin users. In v0.3, PRP Admin users will use inline editing to proof and correct all translations directly in-screen — that is the intended review mechanism, not a dev task.
- **Translation proof-checking is not a dev task.** FR/ES copy on both admin and applicant-facing screens will be reviewed and corrected by users via the v0.3 PRP Admin inline editing tool.
- **ENR terminology rename** and the other stakeholder items listed above are sequenced for this weekend but not yet shipped as of this writing.
