# v0.29 Release Notes — LA 2028 Press Registration Portal

**Release window:** 2–4 May 2026
**Audience:** IOC, LA28 ACR, Deloitte delivery team

---

## Shipped

### EoI Application Form (Applicant-facing)

- **ENR programming type message now conditional.** The advisory about ENR programming type on the Publication tab only appears for Non-Media Rights-Holder organisations. Previously it showed for all applicants regardless of org type.
- **Phone number auto-formatting.** Org phone and contact cell phone fields now format automatically as the user types, using international standards (libphonenumber). Works for all country formats; no manual formatting required.
- **Spanish (ES) language option confirmed live.** The ES toggle on the public EoI form was present in earlier builds but not deploying correctly. Now confirmed working alongside EN and FR.
- **Multi-sport breakdown prompt.** A contextual message now appears under the "About your coverage" field when an applicant requests Es or EPs accreditation, prompting them to enter a per-sport breakdown (e.g. "2 EPs Athletics + 2 EPs Swimming"). Shown only when Es or EPs count > 0.
- **NOC selection copy updated.** Help text now reads "Apply to the NOC of the country where your organisation is based." Tier 2 / home-base language removed.
- **Reserved-organisation gate removed.** Previously, EoI submissions from certain email domains were hard-blocked with a redirect before reaching the NOC. Behaviour changed: the EoI is accepted and submitted normally. The reserved-org list and IOC-Direct list remain for admin use; they are no longer an applicant gate.
- **Preferred language persists.** The language selected by the applicant at time of submission is stored on the application record. Used to serve receipts and status emails in the applicant's preferred language.
- **Data protection notice added.** A placeholder notice now appears at the bottom of the EoI form. Exact copy pending OCOG/legal review; placeholder text is clearly marked.
- **RUS/BLR soft flag.** EoI submissions from organisations in Russia (RU) or Belarus (BY) are accepted and submitted normally, but a `country_flagged` field is set on the organisation record. This surfaced in the NOC queue for review. EOR (Refugee Olympic Team) is country-less and bypasses this check entirely.

### Admin — ENR Nominations

- **"Non-Media Rights-Holders" terminology throughout.** All admin copy previously referring to "Extended Non-Rights Broadcasters" or "Extended Non-Rights" has been replaced with "Non-Media Rights-Holders." The ENR accreditation code is unchanged.
- **ENR submission limit is now a soft warning.** The 3-organisation cap per NOC was previously a hard block. It is now a warning banner — NOC admins can add and submit more than 3 ENR nominations; the IOC decides how many to grant.

### Admin — EOR (Refugee Olympic Team)

- **EOR NOC added.** An `EOR` row has been inserted into `noc_quotas` so the Refugee Olympic Team can be managed through the portal. EOR does not appear in the public EoI NOC dropdown. EOR admins are not yet provisioned — to be done when account setup is confirmed.

### Admin — Deduplication

- **Personal email domains excluded from dedup signals.** Gmail, Outlook, Hotmail, Yahoo, iCloud, AOL, Proton, and other common personal domains are no longer used as a dedup signal. A match on personal email domain is meaningless as an indicator of duplicate organisations.
- **Freelancers excluded from email-domain dedup.** For freelancer org types, the email domain signal is skipped entirely. Name + country remains the dedup key.

### Admin — Email Service

- **Email service stub.** A `sendEmail(type, payload)` function now handles all transactional email. Supports three types: `eoi_receipt`, `invite`, and `magic_link`. Logs in dev; POSTs to a configurable endpoint in prod. Wired to: magic-link token requests, NOC invite flow, and EoI receipt on submission.
- **EoI receipt sent on submission.** Applicants now receive a receipt email on successful EoI submission. Receipt copy confirms the application is under review by NOC; no further comms from PRP; magic link is for status checking only.
- **`eoi_receipt_sent` audit action.** Audit log records when a receipt is dispatched, including the target email.

### Admin — PbN Excel Re-Import Audit

- **Per-field audit entries on re-import.** When a NOC re-imports a PbN Excel file, any changed allocation field now writes a dedicated audit entry in the format `"PbN re-import: {field} {old} → {new}"`. Previously only a summary entry was written with an incorrect action code.
- **`excel_reimport` audit action.** New enum value correctly identifies re-import events in the audit trail.

### Admin — IOC-Direct Approval Flow

- **IOC self-approves.** The IOC-Direct allocation flow no longer waits for OCOG approval. On IOC submit, the allocation transitions directly to `ocog_approved`. OCOG retains read-only visibility of IOC-Direct entries in the cross-NOC view, with an "IOC-Direct" badge.
- **Audit log records both submit and approve actions** when IOC submits, actor = ioc_admin.

### Admin Header

- **Role badge removed.** The static role label that appeared in the admin header bar is gone. Cleaner header; role is visible via the user menu instead.
- **Username is now a dropdown.** Clicking the username in the admin header opens a menu showing the user's assigned roles and the Sign Out action.

### Admin Language Preference (NOC)

- **Language toggle in the admin header.** NOC users can switch between EN, FR, and ES from the user menu dropdown.
- **Preference persists across sessions.** Language choice is saved to the user's account in the database and restored automatically on next login.
- **Full translation dictionaries created.** Complete EN, FR, and ES string sets (~130 strings each) covering navigation, queue columns, status labels, action buttons, EoI detail, PbN, ENR, Direct Entry, Audit Trail, and error messages.
- **NOC admin pages wired.** All NOC admin pages (dashboard, EoI queue, queue detail, PbN, ENR, Direct Entry, Audit Trail) now render in the user's preferred language.

## Notes / Known Gaps

- **IOC and OCOG page translations deferred.** The language toggle is live for IOC and OCOG users, but page copy on those portals stays English. NOC is fully wired; IOC/OCOG follow in a subsequent release via PRP Admin inline editing.
- **PRP Admin not in v0.29.** Inline editing, string publishing, and the PRP Admin settings panel are scheduled for v0.3.
- **Translation proof-checking is not a dev task.** FR/ES copy will be reviewed and corrected by users via the v0.3 PRP Admin inline editing tool.
- **A13 / A14 deferred.** Direct Entry enhancements (A13) require investigation of current Direct Entry code before implementation. Contact info editability (A14) requires a design decision on magic-link behaviour when submitter email changes. Both are tracked in TODOS.md.
- **EOR admin accounts not yet provisioned.** EOR quota row exists; user accounts to be created when account setup is confirmed with IOC.
