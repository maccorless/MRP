# v0.29 Release Notes — LA 2028 Press Registration Portal

**Release window:** 2–4 May 2026
**Audience:** IOC, LA28 ACR, Deloitte delivery team

---

## What's New

### Applicant EoI Form

**Full multi-language support.** Applicants can now complete the entire EoI form in English, French, or Spanish. All field labels, placeholders, help text, error messages, and category descriptions switch with the language toggle. The language selected at submission is saved to the application record and used for receipt emails.

**Phone number auto-formatting.** Both the organisation phone and contact mobile fields format automatically as the applicant types, following international numbering standards. Type with a `+` and country code (e.g. `+1`, `+44`, `+33`) to activate formatting. A help note below each phone field explains this.

**Multi-sport breakdown prompt.** When an applicant requests Es or EPs accreditation, a contextual message appears prompting them to describe their per-sport coverage breakdown (e.g. "2 EPs Athletics + 2 EPs Swimming"). Shown only when Es or EPs count is greater than zero.

**NOC selection copy updated.** Help text now reads: "Apply to the NOC of the country where your organisation is based." Previous Tier 2 / home-base language removed.

**Data protection notice.** A placeholder privacy notice now appears on the EoI form. Exact copy is pending OCOG/legal review; the placeholder is clearly marked.

**Reserved-organisation gate removed.** Previously, certain email domains triggered a hard redirect before the EoI reached the NOC. The EoI is now accepted and submitted normally. The reserved-org and IOC-Direct lists remain for admin reference only.

**RUS/BLR soft flag.** EoI submissions from organisations in Russia or Belarus are accepted normally. A `country_flagged` field is set on the organisation record and surfaced in the NOC queue for review. EOR (Refugee Olympic Team) is country-less and bypasses this check.

---

### NOC Admin

**Language toggle.** NOC admins can switch between EN, FR, and ES from the user menu. All NOC admin pages — EoI queue, queue detail, PbN, ENR, Direct Entry, Audit Trail — render in the selected language. The preference is saved to the user account and restored on next login.

**"Non-Media Rights-Holders" terminology.** All admin copy previously reading "Extended Non-Rights Broadcasters" has been replaced with "Non-Media Rights-Holders." The ENR accreditation code is unchanged.

**ENR soft limit.** The 3-organisation cap per NOC was previously a hard block. It is now a soft warning — NOC admins can add and submit more than 3 ENR nominations; the IOC decides how many to grant.

**Personal email domains excluded from dedup.** Gmail, Outlook, Yahoo, iCloud, and other common personal domains are no longer used as a dedup signal — a match on personal email domain is not a meaningful indicator of duplicate organisations. Freelancer org types are also excluded from email-domain dedup entirely.

---

### IOC Admin

**IOC self-approves direct allocations.** The IOC-Direct allocation flow no longer requires OCOG approval. On IOC submit, the allocation moves immediately to approved status. OCOG retains read-only visibility with an "IOC-Direct" badge.

**EOR (Refugee Olympic Team) added.** EOR is now a manageable NOC in the portal. It does not appear in the public EoI dropdown. EOR admin accounts are not yet provisioned — to be confirmed with IOC.

---

### Email Service

**Transactional email stub wired.** A `sendEmail()` service now handles all three email types: EoI receipt, NOC invite, and magic link. EoI receipt emails fire on submission in the applicant's preferred language. The stub logs in dev and POSTs to a configurable endpoint in production. `eoi_receipt_sent` is now recorded in the audit log.

---

### Admin Header

**User menu redesign.** The static role badge in the admin header has been replaced by a username dropdown showing assigned roles and a Sign Out link.

---

## Known Gaps

- IOC and OCOG page translations are not yet wired. The language toggle is visible but copy stays English on those portals. Both will be addressed via the v0.3 PRP Admin inline editing tool.
- PRP Admin (inline string editing, section publishing, settings panel) is not in v0.29. Scheduled for v0.3.
- FR/ES translation proof-checking is not a dev task. Users will review and correct translations via PRP Admin in v0.3.
- Direct Entry enhancements (A13) and contact info editability (A14) are deferred — tracked in TODOS.md.
- EOR admin accounts not yet provisioned.

## Technical Changes

**New database migrations:**
- `0029_eor_noc_code` — inserts EOR row into `noc_quotas`
- `0030_preferred_language` — adds `preferred_language` enum (EN/FR/ES) to `applications`
- `0031_audit_email_reimport` — adds `eoi_receipt_sent` and `excel_reimport` to `audit_action` enum
- `0032_contact_info_updated_at` — adds `contact_info_updated_at` timestamp to `organizations`
- `0033_prp_admin_role` — adds `prp_admin` to `actor_type` enum
- `0034_content_management` — creates `content_strings` and `section_publish_state` tables (v0.3 infrastructure)
- `0037_country_flagged` — adds `country_flagged` boolean to `organizations`
- `0038_previous_contact_email` — adds `previous_contact_email` to `organizations`
- `0039_contact_info_updated_action` — adds `contact_info_updated` to `audit_action` enum
- `0040_admin_preferred_language` — adds `preferred_language` column to `admin_users` (EN/FR/ES, default EN)

**New i18n translation keys:** ~130 strings added across `applyb.org.*`, `applyb.contact.*`, `applyb.acr.*`, `applyb.pub.*`, and `applyb.addl.*` namespaces in EN, FR, and ES. All NOC admin string sets (~130 strings each) covering navigation, queue columns, status labels, action buttons, EoI detail, PbN, ENR, Direct Entry, and Audit Trail.

**PbN re-import audit log:** Per-field entries now written on Excel re-import (`"PbN re-import: {field} {old} → {new}"`). New `excel_reimport` audit action correctly identifies these events.

**Email service:** `src/lib/email.ts` — `sendEmail(type, payload)` with `eoi_receipt`, `invite`, and `magic_link` types. Logs to console in dev; HTTP POST to `EMAIL_SERVICE_URL` env var in production.
