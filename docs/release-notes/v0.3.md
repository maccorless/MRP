# v0.3 Release Notes — LA 2028 Press Registration Portal

**Release window:** TBD (follows v0.29)
**Audience:** IOC, LA28 ACR, Deloitte delivery team

---

## Shipped (carries forward all v0.29 items, plus the following)

### PRP Admin Role

- **New admin role: PRP Admin.** A dedicated role for the team managing the portal itself. PRP Admin users hold a secondary role alongside an existing admin identity (e.g., IOC Admin + PRP Admin). The role grants access to a new PRP Admin section in the nav — invisible to all other roles.
- **Multi-role support.** The user model now supports multiple simultaneous roles per account. Role assignments are managed via a junction table; a user can hold any combination of roles.

### Content Management — Inline Editing

- **Edit any string in-screen.** PRP Admin users see an admin bar at the top of every portal screen. Switching to Draft mode highlights all editable strings with a dashed outline. Clicking any string opens a small popover with the current value and an edit field. Saving writes the draft immediately — no page reload.
- **Section-level Publish.** The admin bar shows the current section name and a Publish button. Publishing writes all draft strings for that section and language to a persistent volume file and flips the section's published state. Other users see the published copy instantly; unpublished drafts are visible only to PRP Admin in preview mode.
- **Per-language editing.** The admin bar includes a language selector (EN / FR / ES). Edits apply to the selected language only. Each section × language combination has its own publish state.
- **Translation proof-checking via the tool.** FR and ES copy (both admin portal and applicant-facing EoI form) will be reviewed and corrected directly in-screen by PRP Admin users. This replaces the dev-time translation review step.

### Content Management — Data Model

- New `content_strings` table: stores draft and published string overrides keyed by `(section, key, language)`.
- New `section_publish_state` table: tracks publish status per `(section, language)` pair.
- Sections covered: `eoi_applicant`, `noc_eoi_queue`, `noc_direct_entry`, `noc_pbn`, `noc_enr`, `noc_help`, `ocog_eoi`, `ocog_pbn`, `ioc_master`, `ioc_enr`, `emails`.

### Runtime String Loading

- At request time the app loads the bundled compile-time `.ts` strings, then merges any published overrides from the volume file on top. Published overrides win. Zero DB hits for regular users.
- PRP Admin preview mode bypasses the volume file and queries the DB directly so draft edits are visible before publishing.

### PRP Admin Settings Panel

- **Date management.** Re-open EoI windows; set or extend PbN deadlines per NOC or IF.
- **Country eligibility list.** Add or remove countries from the ineligible-submission list (RUS, BLR, and others as directed).
- **Feature flags.** Toggle feature flags from the admin panel. All changes are audit-logged.
- **IOC-Direct reserved-org list.** Add and remove organisations from the IOC-Direct list without a code deploy.

---

## DB Changes

- Carries forward all v0.29 migrations.
- **`content_strings` table** — section, key, language, value, status (draft | published), updated_at, updated_by.
- **`section_publish_state` table** — section, language, status, published_at, published_by.
- **`user_roles` junction table** — supports many-to-many user-to-role assignments.
- **`prp_admin` added to `actorTypeEnum`** — audit log support for PRP Admin actions.

---

## Notes

- PRP Admin screens are English-only in the admin bar UI. The strings being *edited* can be any language.
- The persistent volume for published strings is provisioned on Railway at `/data/i18n/{lang}.json`. If the volume file is absent the app falls back to bundled strings silently.
- IOC and OCOG portal page translations (toggle exists, copy still English from v0.29) will be corrected by PRP Admin users via the inline editor rather than a dev deploy.
